# ETH Watchtower TUI

ETH Watchtower TUI is a real-time terminal user interface for monitoring Ethereum contract deployments, analyzing risks, and detecting suspicious patterns. It processes log entries from a JSONL file (generated by a backend watcher) and presents them in an interactive dashboard.

## Features

*   **Real-time Monitoring:** Live updates of new contract deployments and events.
*   **Risk Analysis:** Visual risk scores with color-coded indicators (Safe, Low, Medium, High, Critical).
*   **Detailed Inspection:** Drill down into specific events to see contract details, transaction hashes, and decoded input data.
*   **On-Chain Data Fetching:** Fetches live data (balance, code size, gas used) from Ethereum RPCs.
*   **Filtering & Search:**
    *   Filter by specific flags (e.g., "Mintable", "ProxyContract").
    *   Filter by token type (ERC20, ERC721, etc.).
    *   Filter by time range (since/until).
    *   Free-text search for contracts, deployers, and transaction hashes.
*   **Heatmap Visualization:** Visualizes risk distribution over block ranges.
*   **Statistics Dashboard:** Overview of total events, unique contracts, risk distribution, and top flags.
*   **Watchlist & Pinning:** Pin important contracts or watch specific deployers for future activity.
*   **Command Palette:** Quickly access all features via a searchable command palette (`Ctrl+P`).
*   **Cross-Platform:** Runs on Linux, macOS, and Windows.

## Installation

### Prerequisites

*   Go 1.24 or higher
*   Make (optional, for building)

### Building from Source

1.  Clone the repository:
    ```bash
    git clone https://github.com/yourusername/eth-watchtower-tui.git
    cd eth-watchtower-tui
    ```

2.  Build the binary using `make`:
    ```bash
    make build
    ```
    This will create an `eth-watchtower-tui` binary in the current directory.

3.  (Optional) Cross-compile for other platforms:
    ```bash
    make dist
    ```

## Usage

Run the application by providing the path to your log file:

```bash
./eth-watchtower-tui [flags] [path/to/eth-watchtower.jsonl]
```

If no file path is provided, it defaults to `eth-watchtower.jsonl` in the current directory.

### Flags

*   `--reset-state`: Reset the application state and read the log file from the beginning.
*   `--init-config`: Generate a default `config.json` file.
*   `--since <duration/time>`: Filter logs since a specific time (e.g., `1h`, `2023-10-27T10:00:00Z`).
*   `--until <duration/time>`: Filter logs until a specific time.
*   `--min-risk <int>`: Minimum risk score to display (default: 10).
*   `--max-risk <int>`: Maximum risk score to display (default: 300).
*   `--version`: Print version information and exit.

## Configuration

The application can be configured via a `config.json` file. To generate a default configuration:

```bash
./eth-watchtower-tui --init-config
```

### Configuration Options

*   `logFilePath`: Path to the input JSONL log file.
*   `rpcUrls`: List of Ethereum RPC URLs for fetching on-chain data.
*   `minRiskScore` / `maxRiskScore`: Default risk score filters.
*   `riskColors`: Customize the colors for different risk levels.
*   `defaultSidePaneWidth`: Default width of the side panel.
*   `stateFilePath`, `reviewedFilePath`, etc.: Paths for storing application state.

Example `config.json`:
```json
{
  "logFilePath": "eth-watchtower.jsonl",
  "rpcUrls": [
    "https://eth.llamarpc.com",
    "https://rpc.ankr.com/eth"
  ],
  "minRiskScore": 10,
  "riskColors": {
    "critical": "#FF0000",
    "high": "#FFA500",
    "medium": "#FFFF00",
    "low": "#FFFACD",
    "safe": "#00FF00"
  }
}
```
